Deployment	— контроллер, который управляет созданием и обновлением набора подов (pods).
	- количество Pods не привязанных к Nodes
	- для разворачивания Pods на определённой Nodes, используются:
		- Seletors — это механизм, который позволяет выбирать и фильтровать объекты на основе значений, указанных в метках (labels). Метки — это пары «ключ-значение», которые назначаются объектам (например, подам, сервисам) и указывают идентифицирующие атрибуты. Селекторы помогают идентифицировать и управлять конкретными группами ресурсов в кластере. 
		- Label — это пары ключ-значение, которые можно прикрепить к любому объекту Kubernetes, таким как поды (Pods), сервисы (Services), деплойменты (Deployments) и многие другие.
		- Affinity — это механизм, который позволяет управлять размещением подов (pods) на узлах кластера на основе определённых правил или предпочтений. Это помогает оптимизировать использование ресурсов, обеспечить надёжность и управлять топологией развёртывания приложений.
		- Tolerations — специальные параметры, которые назначаются подам и позволяют им игнорировать определённые taints узлов. Это как «разрешение» для пода быть размещённым на узле с определённым taint. Taints — метки, которые наносятся на узлы и служат для ограничения размещения подов. Если узел имеет taint, поды не будут размещены на этом узле, если они не имеют соответствующих tolerations.
	
Service		- подключение к контейнеру Pod.
Route 	(OpenShift)	- для внешнего API, который ведёт на Service, который ведёт на Pod.
Ingress (k8s)		- для внешнего API, который ведёт на Service, который ведёт на Pod.
ConfigMap	— это ресурс, предназначенный для хранения неконфиденциальных данных конфигурации в виде пар «ключ-значение». ConfigMap позволяет отделить конфигурационные данные от образов контейнеров, что облегчает управление и обновление конфигурации без необходимости пересборки контейнеров. Эти данные могут включать: 
	...параметры конфигурации;
	...настройки окружения ENV;
	...конфигурационные файлы и другие данные, которые необходимы приложению для корректной работы.
	...API address.
Secret		— это объект, предназначенный для безопасного хранения конфиденциальной информации. base64. Он позволяет отделить чувствительные данные от кода приложения и конфигурационных файлов, что упрощает управление безопасностью и снижает риск утечек. Примеры данных, которые хранятся в Secret:
	...пароли для доступа к базе данных;
	...токены API для сторонних сервисов;
	...ключи SSH для доступа к удалённому серверу;
	...сертификаты TLS для HTTPS-коммуникации;
	...токены OAuth для аутентификации сервисов.

Istio в Kubernetes	— это специализированный инфраструктурный слой, который берёт на себя задачи управления сетевым взаимодействием между сервисами. Это "менеджер трафика" и "система безопасности" для всего, что происходит внутри кластера. Управляет трафиком входящим и трафиком между микросервисами. Это sevice-mash, комплексный framework для управления сетью. 4 уровень (TCP). mTLS. Сам Istio.
Ingress (k8s)		- это "шлюз", "входная дверь" в кластер. Знаимается маршрутизацией HTTP/HTTPS трафика. 7 уровень. TLS. Nginx, Traefik, HAProxy
