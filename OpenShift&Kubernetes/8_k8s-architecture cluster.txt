		master node
			control plane ("головной мозг" кластера., может быть несколько)
				kube-api	— компонент кластера Kubernetes, который предоставляет API-интерфейс для управления ресурсами кластера. Это единственный компонент, с которым напрямую взаимодействуют все остальные части системы и внешние клиенты, такие как утилита kubectl или хранилище etcd. API-интерфейс — это REST-интерфейс, который обрабатывает запросы на создание, обновление и удаление ресурсов (подов, сервисов и т. д.). 
				etcd		- распределённое хранилище данных (key-value store). Это компонент архитектуры Kubernetes, который хранит состояние кластера, включая конфигурации, определения объектов и чувствительные данные (secrets, ConfigMaps, токены ServiceAccount и т. п.). 
				scheduler	- планировщик подов (Pods), который определяет оптимальную ноду для размещения нового пода в кластере. Работает как часть плоскости управления (control plane).  Основная задача планировщика — оптимизировать размещение подов, учитывая доступные ресурсы на узлах, требования каждого пода и другие факторы. 
		worker node (где запускаются поды приложения)

	ОГРАНИЧИТЕЛИ КЛАСТЕРА:
		namespace (изолированная область)
			kube system (control plane) 	— это плоскость управления k8s, расположены основные объекты kubernetes.
			nodes — рабочие узлы:
				Daemonset		— это контроллер, который гарантирует, что на каждом узле кластера запущен один экземпляр определённого пода. Это позволяет запускать системные сервисы и агентов, которые должны работать повсеместно, независимо от нагрузки или числа приложений.
				Secret
				Service account		— это учётная запись службы, которая используется процессами или приложениями, работающими внутри подов (pods), для аутентификации и взаимодействия с сервером API Kubernetes. В отличие от учётных записей пользователей, которые обычно связаны с людьми, Service Accounts предназначены для нечеловеческого доступа. 
		Project (изолированная область Openshift) 	- это namespace приложений.
