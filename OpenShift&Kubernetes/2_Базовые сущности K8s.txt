===== Pod =====	самая простая и примитивная сущность/объект Cluster.
		императивный подход, вносить новые объекты в кластер. В команде передаём параметры.
.yaml	-	декларативный подход, описание требуемого состояния объекта. Удобно щаблонизировать.
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
   containers:
     - name: nginx
       images: *****
       ports:
         - containerPort: 80
kebectl run apply nginx --image=nginx:1.14.2	-	императивный подход
kubectl apply -f simplie-pod.yaml		-	декларативный подход. Переводит .yaml в формат .json и сообщает в API Cluster
oc apply -f simplie-pod.yaml			-	декларативный подход


===== контроллеры ========
ReplicaSet	-	контролирует (минимальное) количество реплик у Pod, порождает создание Pods;
Deployments	-	для разворачивания приложения, декларативно беря из .yaml: количество реплик, стратегия разворачивания, порождает replicaset.


===== cattle and pats ===== подходы/стратегия к управлению ресурсами.
Pats	-	любимчик, 
Cattle	-	стадо. Безымянные объекты.
Следит оркестратор за стадом.
