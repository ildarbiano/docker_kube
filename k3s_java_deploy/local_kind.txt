Kind (Kubernetes IN Docker) 		- это легковесный инструмент для запуска локальных Kubernetes кластеров используя Docker контейнеры.
🐳 Kubernetes IN Docker			- запускает ноды Kubernetes как Docker контейнеры
⚡ Быстрый запуск			- кластер создается за секунды
🎯 Для разработки/тестирования		- идеален для локальной разработки
🔧 Простая настройка			- минимальная конфигурация
🆓 Бесплатный				- открытый исходный код

Сравнение: 	Kind 		vs 	Minikube
Характеристика	Kind			Minikube
Запуск		Секунды			Минуты
Память		~300MB на ноду		~2GB+
Стабильность	Высокая			Частые зависания
Архитектура	Мульти-нодный кластер	Одна VM/контейнер
Использование	Разработка, CI/CD	Обучение, демо
========= Почему рекомендую Kind для вас:
У вас мало RAM (3.8GB) 		- Kind потребляет меньше
Частые зависания Minikube 	- Kind стабильнее
Быстрый перезапуск 		- если что-то сломается
Идеален для обучения 		- настоящий Kubernetes

========= УСТАНОВКА kind ===================================
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-linux-amd64
chmod +x ./kind						# Сделать исполняемым
sudo mv ./kind /usr/local/bin/				# Установить в систему
sudo apt update && sudo apt install -y docker.io	# Обновляем Docker

========= Создать кластер ==================================
kind create cluster --name k7s-project
Проблема в версии Docker! Kind требует Docker 20.10+, а у вас 18.09.7.
